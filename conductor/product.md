# 产品定义 - AEngine

## 产品愿景
AEngine 是一款专为游戏引擎开发者设计的高性能、高度模块化实时渲染器。它采用 C++ 开发，强调严谨的工程化和多线程架构，旨在通过插件和子系统机制，提供一个灵活且健壮的框架，用于前沿图形学算法的研究与实现。

## 目标用户
- **游戏引擎开发者**：需要一个高度解耦、架构清晰的平台，用于验证自定义渲染算法、开发图形插件或集成特定的渲染子系统。

## 核心架构特性
- **子系统 (Subsystem) 架构**：
    - 参考 **Unreal Engine** 的 Subsystem 机制，将核心功能（如物理、音频、输入、渲染核心）抽象为独立的子系统，支持按需启动、停止和管理自动化的生命周期。
- **插件 (Plugin) 机制**：
    - 借鉴 **Unreal Engine** 和 **Godot** 的插件设计，所有的渲染特性（如 GI 算法、特定的 Pass、新材质模型）均可通过动态链接库（DLL/SO）形式加载，实现真正的热插拔、独立开发和分发。
- **高性能多线程**：
    - **任务系统 (Task System)**：参考 **Naughty Dog** 或 **Godot** 的设计，基于纤程 (Fiber) 或线程池的任务调度，最大化利用多核 CPU。
    - **并行渲染**：参考 **CryEngine** 和 **Unreal Engine** 的渲染线程架构，实现渲染指令生成的并行化 (Command Buffer Generation)，将逻辑更新与渲染提交分离。
- **多 API 渲染抽象层 (RHI)**：提供极简 RHI 封装，支持 OpenGL、Vulkan 和 DirectX 的无缝切换，同时暴露底层 API 能力以供高级调优。

## 渲染特性（模块化）
- **可自定义渲染模型**：内置 PBR 插件，同时开放接口支持完全自定义的非 PBR 渲染管线（类似于 **Godot** 的可编程管线或 **Unity** 的 SRP）。
- **模块化光线追踪**：光线追踪功能作为独立插件提供，支持 DXR/Vulkan Ray Tracing 的灵活配置。
- **灵活的全局光照 (GI)**：DDGI、SSGI 等算法均以插件形式存在，支持运行时切换。

## 用户界面 (基于 ImGui)
- **场景图与属性检查器**：可视化管理场景实体及子系统状态。
- **资源与插件管理器**：可视化加载、卸载插件，浏览和管理资源。
- **管线编辑器**：可视化构建 Frame Graph，编排不同插件提供的渲染 Pass。

## 关键技术目标
- **极致轻量化与工程化**：核心框架保持最小化，强依赖管理，清晰的构建流程（CMake），确保易于集成和维护。
