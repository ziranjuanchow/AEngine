# Track 规格说明 - 场景大纲与层级管理 (Scene Editor & Hierarchy Management)

## 1. 概述
本 Track 旨在为 AEngine 提供一个交互式的场景编辑器。通过实现层级化的场景树管理、大纲视图和属性检查器，使用户能够动态地组织场景、修改物体属性并追加导入新资源，从而从“单模型演示”进化为“多物体场景编辑器”。

## 2. 功能需求
### 2.1 场景架构增强 (Scene Infrastructure)
- **全局场景树**：维护一个单例或全局可见的 `Root` 节点（FSceneNode）。
- **追加式加载**：更新 `AssetLoader` 的集成逻辑，支持将加载的模型挂载到指定的父节点下，而不是替换整个场景。

### 2.2 场景大纲面板 (Scene Outliner)
- **层级显示**：使用 ImGui 的 `TreeNode` 递归渲染场景树。
- **节点交互**：
    - **选中 (Selection)**：支持单击选中单个节点，结合 Ctrl 支持多选。
    - **管理**：右键菜单支持“重命名”和“删除”节点。
    - **可见性**：每一行提供可见性开关（眼睛图标）。

### 2.3 属性检查器面板 (Inspector)
- **变换编辑**：显示并允许修改选中的 `FSceneNode` 的 Position, Rotation, Scale。
- **材质管理**：
    - 实时编辑关联的 `FStandardPBRMaterial` 参数。
    - 支持显示材质元数据（如贴图路径）。
- **统计信息**：显示选中物体的顶点数和面数。

## 3. 交互逻辑
- **导入流**：点击 "Load Model" 时，如果当前有选中节点，新模型将作为其子节点导入；若无选中，则默认挂载至 `Root`。

## 4. 技术约束
- 场景管理必须基于内存运行，本阶段不涉及序列化。
- 必须保持渲染循环对整个场景树的遍历效率。

## 5. 验收标准
- 能够在场景中同时显示多个不同的模型。
- 通过大纲面板删除节点后，屏幕上的物体立即消失且内存正确释放。
- 能够通过属性面板将某个物体的缩放改为 2 倍，且其子物体同步放大。

## 6. 超出范围
- 场景保存与加载 (Save/Load)。
- 3D 变换操作轴 (Gizmos) - 本 Track 仅支持数值输入。
